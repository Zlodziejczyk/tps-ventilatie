# Quick Task 260628-4d3: TIER 2 — ServiceHero redesign (walls of text)

## Boundary

Owner preview review (2026-06-28) defect **#4**: service sub-pages
(`/diensten/.../onderhoud`, `/airconditioning/reparatie-storing`, …) read as
**walls of text** — `components/ServiceHero.tsx` dumps H1 + the full 120-word+
intro as one block, and the right half of the hero is an empty faint glow
placeholder. This task **redesigns `ServiceHero`** for scannability + hierarchy
and fills the right column. **Code+design only, zero new assets** (TIER 2).

`ServiceHero` is a **shared template** component rendered on BOTH service pages
(`app/diensten/[pillar]/[service]/page.tsx`) AND pillar pages
(`app/diensten/[pillar]/page.tsx`). The redesign must therefore be **generic
across every node** — driven only by `PageBase` fields + `SITE`.

Design language: load `Skill(sketch-findings-tps-ventilatie)` — "Atmospheric
Clarity, **engineered**" (signature-gradient, glass card, `air-pulse` live
"direct beschikbaar" dot, soft cool-tinted shadow, no 1px section borders, no
#000, MD3 tokens only).

## Why a non-photo trust card (not an image)

`public/images/` has wtw / mv / work photos but **none for airco/warmtepompen**,
and the template is shared across all 4 pillars + 17 services. A per-service
photo would be asset-blocked (that's TIER 3). So the right column is a **glass
key-facts / trust card** built from `SITE` + node data — always correct, asset
free, and conversion-forward (the sketch lever "speed-to-contact").

## Hard constraints (carried from phase 04 / STATE.md — UNCHANGED)

- Do **NOT** `git push origin main` or `vercel --prod`. `main` is ~143 ahead of
  origin and maps to Vercel **production** — a push/prod-deploy IS the launch.
  Deploy a **preview** only via `vercel deploy --yes` (owner-run).
- Do **NOT** build locally — the OneDrive mount deadlocks `tsx`/`next build`.
  Validate on **Vercel CI** via the preview deploy.
- Run **inline** (no subagents / no worktrees) — OneDrive hangs spawned
  executors (GSD SDK is not installed in this repo either).
- **Do not rewrite copy.** The intro/h1 text is owner-approved data in
  `lib/services/*.ts`. The lead/supporting split is a pure **presentation**
  transform of the existing string — no content authored or changed.
- Keep it a **Server Component** (no `"use client"`); reuse the existing cheap
  `AnimateOnScroll` motion tier only (CWV / SEO-10 mobile-INP safe).

## Task — Redesign `components/ServiceHero.tsx`

- **files:** `components/ServiceHero.tsx` (single file; internal non-exported
  `HeroFacts` sub-component per the repo idiom)
- **action:**
  1. **Two-column layout** (`grid lg:grid-cols-12`, `items-start`): text left
     (`lg:col-span-7`), trust card right (`lg:col-span-5`). Stacks on mobile.
  2. **Break the wall of text** — `splitLead()` pure helper splits the intro
     into a **lead sentence** (first sentence, emphasized `text-xl/2xl
     font-medium text-on-surface`) + **supporting copy** (remainder,
     `text-on-surface-variant font-light`). Robust split: first `.?!` followed
     by whitespace+capital with a ≥30-char min lead (won't break on "bijv.").
     Falls back to navDescription when intro is empty (keep D-06 behavior); if
     no boundary, whole text renders as the lead (no regression).
  3. **Keep** the icon chip + H1; **left CTA row** = primary "Offerte
     aanvragen" (signature-gradient, → /contact) + secondary "Bel
     {phoneDisplay}" (tonal `bg-surface-container-high`, `tel:`).
  4. **`HeroFacts` trust card** (fills the right column): glass
     `bg-surface-container-lowest`, engineered `signature-gradient` top-accent
     bar, soft teal shadow; a live **`air-pulse`** "Direct beschikbaar voor
     advies" dot (tertiary/green); 3 universally-true fact rows from `SITE`
     (`distance` → Werkgebied {city} + {serviceRadiusKm} km; `schedule` →
     Bereikbaar Ma–za 08:00–17:30, ook op afspraak; `verified_user` →
     Vakkundig & vrijblijvend, eerlijke prijs); one **WhatsApp** action
     (`bg-[#25D366]`, `SITE.whatsappUrl`). No specific BRL/cert claims here
     (those are the TIER 3 trust band) — keep facts true for every node.
  5. Tokens/icons only via MD3 vars + the `<Icon>` wrapper; contact data only
     from `SITE`. No 1px section borders, no #000.
- **verify (on Vercel CI preview — owner-run):**
  - TypeScript compiles clean (no local build); 32 static pages generate.
  - A service page (e.g. `/diensten/warmtepompen/onderhoud`) shows: H1 → bold
    lead sentence → lighter supporting paragraph → CTA row, with the trust card
    beside it on desktop / stacked on mobile; pulse animates (and is static
    under `prefers-reduced-motion`).
  - `grep -n '"use client"' components/ServiceHero.tsx` → nothing (still SC).
- **done:** service/pillar heroes read as a scannable, hierarchical, two-column
  unit with a filled right column; no wall of text; zero new assets; owner can
  re-check on the fresh preview.
