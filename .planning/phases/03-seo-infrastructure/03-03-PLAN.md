# 03-03 — JSON-LD component + structured-data builders

## Objective

Provide the reusable server-side `JsonLd` injector + the typed builders that turn `SITE` + a
taxonomy node into valid Schema.org structured data: a site-wide `HVACBusiness` node and per-page
`Service`, `BreadcrumbList`, `FAQPage` (D-04). Server-rendered, zero client JS, XSS-safe, and
**no `aggregateRating`/`review`** (reserved for Phase 4 / CONT-08). Regio signals (SEO-06) ride
in via `areaServed`/`serviceAreas`.

Purpose: this is the structured-data layer the layout (sitewide) and the routes (per-page)
consume. Must validate in Google's Rich Results Test (success criterion 3).
Output: `lib/seo/jsonld.tsx` (`JsonLd`, `businessJsonLd`, `serviceJsonLd`, `breadcrumbJsonLd`, `faqJsonLd`).

## must_haves

- `JsonLd({ data })` renders `<script type="application/ld+json">` with `JSON.stringify(data)`
  **escaping `<` → `<`** (prevents `</script>` breakout). Server component, no `"use client"`.
- `businessJsonLd()` → `@type "HVACBusiness"`, stable `@id = ${origin}/#business`, built entirely
  from `SITE`: `name`, `address` (PostalAddress: street/postcode/city/country), `geo`
  (GeoCoordinates), `telephone`, `email`, `url` = origin, `areaServed` = `GeoCircle` (geoMidpoint
  = geo, geoRadius 60000) PLUS `serviceAreas` as `Place[]`, `priceRange`, `image`/`logo`. Carries
  **NO** `aggregateRating` and **NO** `review` key. Omits `sameAs` while owner GBP/social URLs are
  pending (A-3) — no empty/placeholder URLs.
- `serviceJsonLd(node)` → `@type "Service"`, `name`/`serviceType` from `navTitle`/`primaryKeyword`,
  `provider` = `{ "@id": ${origin}/#business }`, `areaServed` from `SITE.serviceAreas`. For
  pillar + service nodes.
- `breadcrumbJsonLd(node)` → `@type "BreadcrumbList"` from `trailFor(node)`, each `item` an
  absolute URL via `absoluteUrl(crumb.href)`, `position` 1-indexed.
- `faqJsonLd(node)` → `@type "FAQPage"` from `node.content.faqs`, returns **null when faqs is
  empty** (so it only renders once Phase 4 fills FAQs).
- All builders pure; all business data from `SITE` (no hardcoding); origin via `CANONICAL_ORIGIN`/`absoluteUrl`.
