# 03-01 — SEO foundation: canonical origin + GSC field + indexing policy

## Objective

Lay the single-source foundation every other Phase-3 plan reads: the canonical origin constant
(D-01, live-confirmed apex), the Google Search Console verification value (D-06), and the
**one** indexing-policy helper that decides sitemap membership + per-page index directive
(D-02). Mirrors the `urlFor()` single-source pattern so canonical / sitemap / robots can never
drift. Pure data + pure functions — no rendering, no new dependencies.

Purpose: nothing downstream (sitemap, robots, metadata builder, JSON-LD, layout) can compute an
absolute URL or an index decision until these exist. This is Wave 1, the root of the graph.
Output: `CANONICAL_ORIGIN` + `GOOGLE_SITE_VERIFICATION` in `lib/constants.ts`; new
`lib/seo/policy.ts` exporting `isIndexable`, `absoluteUrl`, `sitemapEntries`.

## must_haves

- `lib/constants.ts` exports `CANONICAL_ORIGIN === "https://tpsventilatie.nl"` (apex, D-01) and
  `GOOGLE_SITE_VERIFICATION` sourced from `process.env.NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION`
  (empty string when unset). `SITE` itself is left structurally unchanged (siblings added, not a
  rewrite of the `as const` object).
- `lib/seo/policy.ts` `isIndexable(node)` returns: `true` for every `static` node EXCEPT
  `pathSegment === "privacy-beleid"`; `node.status === "published"` for `hub`/`pillar`/`service`.
  Given the current all-draft registry this yields exactly the 4 static content pages indexable
  (`/`, `/tarieven`, `/over-ons`, `/contact`).
- `absoluteUrl(path)` returns `CANONICAL_ORIGIN + "/"` for `"/"` and `CANONICAL_ORIGIN + path`
  otherwise (no double slash, no trailing slash on non-root — honors P1 D-03 / `trailingSlash:false`).
- `sitemapEntries()` returns one `{ url, lastModified? }` per indexable node, `url` absolute via
  `absoluteUrl(urlFor(node))`; nothing draft, no `privacy-beleid`.
- No hardcoded business data, no `"use client"`, reads `PAGES`/`urlFor` from the registry.
