# 03-07 — Static-page metadata via the shared builder

## Objective

Route every static page's `metadata` through `buildMetadata` (03-02) so each emits an absolute
self-canonical + OG/Twitter + the correct per-page `robots` directive (D-05, D-02). The 4 content
pages (home, tarieven, over-ons, contact) become indexable now; `privacy-beleid` and the draft hub
`/diensten` emit `noindex,follow`. Home currently has **no** metadata export — this adds one.

Purpose: close the canonical/OG/robots gap on the pages real users + Google already see, using the
same builder as the dynamic routes so nothing drifts. Output: 6 updated page metadata exports.

## must_haves

- Home `app/page.tsx` gains `export const metadata = buildMetadata(findBySlug("/")!)` (it had none;
  no longer relies solely on the layout default) → indexable, absolute canonical `https://tpsventilatie.nl/`.
- `tarieven`, `over-ons`, `contact` `metadata` exports replaced with `buildMetadata(findBySlug("/<seg>")!)`
  → indexable, absolute self-canonical + OG/Twitter.
- `privacy-beleid` → `buildMetadata(findBySlug("/privacy-beleid")!)` → `robots` `noindex,follow`
  (policy: static privacy-beleid is the one non-indexable static).
- Hub `app/diensten/page.tsx` → `buildMetadata(findBySlug("/diensten")!)` → `noindex,follow` while
  the hub node is draft (auto-flips when Phase 4 publishes it).
- Each page keeps its existing visible content + any non-metadata exports untouched; titles continue
  to come from each node's `content.metaTitle` (already klimaattechniek-branded in the registry).

<threat_model>
Static-export page metadata only — build-time, typed node data, no user input, no secrets, no
runtime. `findBySlug(...)!` is a known-safe lookup (the slugs are registry constants; a missing
node would fail `tsc`/build, not produce an injection). Output is metadata serialized by Next (no
`dangerouslySetInnerHTML`). STRIDE surface: none meaningful. Residual risk: **none (LOW)**.
Supply-chain: no new packages.
</threat_model>
