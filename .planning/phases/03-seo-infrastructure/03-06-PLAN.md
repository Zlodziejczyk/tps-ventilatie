# 03-06 — Dynamic route metadata + per-page JSON-LD

## Objective

Widen the two dynamic routes' `generateMetadata` from the Phase-2 title/description seam (D-13)
to the full canonical + OG/Twitter + per-page `robots` via the shared `buildMetadata` (03-02),
and inject per-page `Service` + `BreadcrumbList` + (when present) `FAQPage` JSON-LD via the 03-03
builders (D-04, D-05). These 21 routes (4 pillar + 17 service) carry `noindex,follow` while draft
and flip to indexed automatically when Phase 4 publishes them — no rework here.

Purpose: the service-surface pages are the SEO core; this is where their canonical + structured
data land. Output: updated `app/diensten/[pillar]/page.tsx` + `app/diensten/[pillar]/[service]/page.tsx`.

## must_haves

- Both routes' `generateMetadata` resolves the node (`findPillar` / `findService`, awaiting
  `params` — Next 16, unchanged) and returns `buildMetadata(node)` (absolute self-canonical + OG/
  Twitter + `robots` from policy). When the node is undefined, return a minimal safe object (the
  `notFound()` in the page body already handles the 404; `dynamicParams=false` guards the param set).
- Pillar page renders `<JsonLd data={serviceJsonLd(node)} />` + `<JsonLd data={breadcrumbJsonLd(node)} />`
  and, when `node.content.faqs.length > 0`, `<JsonLd data={faqJsonLd(node)!} />`.
- Sub-service page renders the same three (Service + BreadcrumbList + conditional FAQPage).
- `export const dynamicParams = false` + `await params` preserved; both stay server components
  with cheap-motion-only (no new client imports); existing page layout/markup unchanged otherwise.
