# 03-04 — Programmatic sitemap + robots

## Objective

Generate `/sitemap.xml` and `/robots.txt` from the taxonomy via the 03-01 policy helper, both
`force-static` so they build under `output: export` (SEO-01, SEO-02). The sitemap lists only
indexable pages (the 4 statics now; service-surface pages auto-join as Phase 4 publishes them).
Robots is open + AI-crawler-friendly with an absolute sitemap pointer (D-03).

Purpose: the two crawl-control files Google + AI crawlers read first — and the no-drift guarantee
(both derive from `sitemapEntries`/`isIndexable`, never a hand-maintained list).
Output: `app/sitemap.ts`, `app/robots.ts`.

## must_haves

- `app/sitemap.ts`: `export const dynamic = "force-static";` and a default function returning
  `MetadataRoute.Sitemap` = `sitemapEntries()` (from 03-01). Resolves at build to `/sitemap.xml`
  with exactly the 4 indexable absolute apex URLs; no drafts, no `/privacy-beleid`, no `/diensten/*`.
- `app/robots.ts`: `export const dynamic = "force-static";` and a default function returning
  `MetadataRoute.Robots` with `rules` = `*` allow `/` PLUS explicit allow entries for the AI bots
  (GPTBot, ClaudeBot, OAI-SearchBot, PerplexityBot, Google-Extended, CCBot), **no `disallow`**
  anywhere (noindex pages stay crawlable so the per-page directive is seen, D-03); `sitemap` =
  `absoluteUrl("/sitemap.xml")`; `host` = `CANONICAL_ORIGIN`.
- Neither file hardcodes a URL list or origin — both read the 03-01 helpers.
