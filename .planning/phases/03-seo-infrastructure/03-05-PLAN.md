# 03-05 — Root layout: rebrand + metadataBase + sitewide JSON-LD + analytics + GSC

## Objective

Fix the root document head and chrome (D-01/D-04/D-05/D-06): set `metadataBase` to the apex,
rebrand the stale "TPS Ventilatie" title/template to "TPS klimaattechniek", drop the deprecated
`keywords` meta, add default OG/Twitter, render the site-wide `HVACBusiness` JSON-LD, wire
cookieless Vercel Analytics + Speed Insights, and emit the GSC verification meta tag.

Purpose: every page inherits this head. It is where the brand cleanup, the metadataBase that makes
relative OG/canonical resolve, the one business JSON-LD node, and measurement all live.
Output: updated `app/layout.tsx`; `@vercel/analytics` + `@vercel/speed-insights` in `package.json`.

## must_haves

- `app/layout.tsx` `metadata.metadataBase = new URL(CANONICAL_ORIGIN)`; `title.default` =
  "TPS klimaattechniek — Airco, warmtepompen & ventilatie in Zoetermeer"; `title.template` =
  "%s | TPS klimaattechniek"; description rebranded to klimaattechniek (regio-bearing, SEO-06);
  the `keywords` array is **removed** (D-05).
- Default `openGraph` (siteName `SITE.name`, `locale: "nl_NL"`, `type: "website"`, default
  `/og-default.jpg`) + `twitter` (`summary_large_image`, same image).
- `verification: { google: GOOGLE_SITE_VERIFICATION }` emitted ONLY when the env value is non-empty.
- Site-wide `<JsonLd data={businessJsonLd()} />` rendered once (HVACBusiness, SEO-03/SEO-06).
- `<Analytics />` (`@vercel/analytics/next`) + `<SpeedInsights />` (`@vercel/speed-insights/next`)
  mounted in `<body>` (D-06, SEO-09); both packages added to `package.json` + lockfile.
- No `"TPS Ventilatie"` string remains in `app/layout.tsx`; `next.config.ts` untouched; layout stays
  a server component (Analytics/SpeedInsights are client components imported into it — allowed).
