# 03-02 — Shared metadata builder + OG image

## Objective

Centralize page metadata in ONE builder (`lib/seo/metadata.ts`) used by both the static-page
`metadata` exports AND the dynamic routes' `generateMetadata` (extending the P2 D-13 seam,
which returns only title/description today) — so `metadataBase`/canonical/OG/Twitter/robots
never drift across the ~27 pages (D-05). Plus the one branded static OG image the builder and
layout default reference (D-05; static export rules out dynamic generation).

Purpose: this is the seam every page's head goes through. Get it right once.
Output: `lib/seo/metadata.ts` (`buildMetadata`, `OG_IMAGE`); `public/og-default.jpg` (1200×630).

## must_haves

- `buildMetadata(node)` returns a Next `Metadata` with: `alternates.canonical` = absolute
  self-canonical via `absoluteUrl(urlFor(node))` (no trailing slash except root, P1 D-03);
  `title` = `node.content.metaTitle`, `description` = `node.content.metaDescription`;
  `openGraph` { `url` = canonical, `title`, `description`, `siteName: SITE.name`, `locale: "nl_NL"`,
  `type: "website"`, `images: [OG_IMAGE]` }; `twitter` { `card: "summary_large_image"`, `title`,
  `description`, `images: [OG_IMAGE]` }; `robots` derived from `isIndexable(node)` →
  `{ index: true, follow: true }` when indexable else `{ index: false, follow: true }` (drafts +
  privacy-beleid stay crawlable so the noindex is actually seen, D-03).
- `OG_IMAGE` constant = `"/og-default.jpg"` (1200×630) exported for the layout default.
- `public/og-default.jpg` exists, 1200×630, branded (seeded from `public/images/hero-ventilatie.jpg`).
- Builder reads `SITE.name` (no hardcoded brand), `CANONICAL_ORIGIN`/`absoluteUrl`/`isIndexable`
  from 03-01. No `"use client"`.
