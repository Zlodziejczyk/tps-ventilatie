<objective>
Make the desktop mega-menu keyboard-operable with correct ARIA (UI-09). Today the Diensten/Tarieven dropdowns open on hover only, with no `aria-expanded`/`aria-haspopup` and no focus-open — so keyboard users get no state and can't open the panel from the keyboard (destinations are still reachable via the pillar pages + mobile menu, so this is an enhancement, not a blocker). Add `aria-haspopup="menu"` + `aria-expanded` to the triggers, open the panel on `onFocus`, and close it on `Escape`, keeping the existing hover/spring behavior intact.

Purpose: Keyboard + screen-reader parity for the primary navigation — an AA interaction-polish item.
Output: ARIA + keyboard open/close on the `Navbar.tsx` dropdown triggers.
</objective>
