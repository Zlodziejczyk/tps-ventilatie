<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → /api/lead | The app's ONLY server endpoint; an unauthenticated, public POST surface — every byte is untrusted |
| /api/lead → GoHighLevel | Server-side forward of validated lead data carrying the secret webhook URL (never client-visible) |
| route → client response | Only generic error codes cross back; no field echoes, no stack traces, no PII |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-secret | Information Disclosure | Webhook URL exposure | mitigate (complete) | Read only `process.env.GHL_WEBHOOK_URL` server-side; verify-grep blocks any `NEXT_PUBLIC_`; 05-06 deletes the old public Vercel var + greps the client bundle to prove absence |
| T-5-payload | Tampering | Malformed/oversized/missing-field bodies | mitigate | `leadSchema.safeParse` (min/max bounds) + malformed-JSON → 400; server-authoritative (client checks are UX only) |
| T-5-spam | DoS / Tampering | Bot spam to GHL | mitigate | Honeypot `website` silent-accept-without-forward + `@upstash/ratelimit` sliding window 5/10min per IP (honeypot remains active even if Upstash unprovisioned) |
| T-5-cors | Spoofing | Cross-origin POST from a third-party page | mitigate (layered) | Honeypot + rate-limit + Zod are the real controls; optional `Origin`/`Content-Type` check as a cheap extra layer (not sole control — Origin is spoofable by non-browser clients, RESEARCH A4) |
| T-5-pii | Information Disclosure | Lead PII leaking via logs/errors | mitigate | Never log the payload; return generic error codes only; GHL is the named AVG processor (privacy text in 05-03) |
| T-5-config | DoS | Route crash if Upstash env missing | mitigate | Limiter constructed only when both Upstash vars present + wrapped so `Redis.fromEnv()` failure can't 500 the route |
</threat_model>
