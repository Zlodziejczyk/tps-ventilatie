# 02-02 — Six IA-05 reusable service components

## Objective

Build the 6 reusable **server components** (IA-05) the route templates compose:
ServiceHero, ServiceSteps, ServiceFAQ, Breadcrumbs, BrandGrid, RelatedServices. These
render `ContentShell`/taxonomy data consistently across all ~22 pages, omit
content-less sections gracefully (D-06), and stay in the cheap CWV-safe motion tier
(D-10 — `AnimateOnScroll`/`StaggerChildren` only; no `SoftAurora`/particles).

All six are server components (no `"use client"`) — they render data, not interactivity.
The only client islands are the existing motion wrappers they reuse. Files are flat in
`components/` to match the established convention (RESEARCH §3 / A-3).

## must_haves

- All 6 components exist as named-export **server** components (no `"use client"`).
- Graceful omit (D-06): ServiceSteps renders nothing when `steps` empty; ServiceFAQ
  renders nothing when `faqs` empty; BrandGrid renders nothing when `brandIds` empty/absent.
- ServiceHero intro falls back to `navDescription` when `content.intro === ""` (D-06).
- BrandGrid renders **text brand chips** (no `<img>`/`<Image>`), and **never** an
  "erkend installateur"/dealer claim (RESEARCH §4).
- Breadcrumbs renders `trailFor(node)` with the last crumb non-linked (current page).
- RelatedServices uses `siblingsOf` (same-pillar, D-11) and reuses `ServiceCard`.
- Design system honored: MD3 tokens only, no `1px solid` separators, no `#000`, Material
  Symbols via `Icon`.
