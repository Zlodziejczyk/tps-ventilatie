# 02-01 тАФ Registry lookups + render helpers

## Objective

Add the small, pure, typed lookup + helper functions that every Phase-2 render unit
(routes, nav, the 6 components) reads from тАФ so the rendering layer never re-implements
taxonomy traversal and `urlFor()` stays the sole href builder (P1 D-03). This is the
data-access foundation; all later waves depend on it.

Scope is `registry.ts` additions only (kept under the documented no-barrel exception)
plus extending the existing `scripts/assert-registry.ts` harness to lock the new
helpers. No route, component, or data-file changes here.

## must_haves

- `pillars()`, `findPillar()`, `childrenOf()`, `findService()`, `siblingsOf()`,
  `brandsForPillar()`, `trailFor()`, `pillarTarievenTab()` exported from `registry.ts`,
  all pure and typed against the `./types` union.
- `trailFor()` returns a `Crumb[]` (`{label,href}`) whose hrefs come from `urlFor()` тАФ
  reusable by Phase 3 for BreadcrumbList JSON-LD (D-13).
- `assert-registry.ts` extended with assertions covering all new helpers; `npx tsx
  scripts/assert-registry.ts` exits 0.
