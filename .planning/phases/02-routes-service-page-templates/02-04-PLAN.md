# 02-04 — Lean /diensten hub + retire DienstenNav

## Objective

Replace the current anchored single-page `/diensten` with a **lean hub** that routes to
the 4 pillars (D-02: Hero → 4 pillar cards → trust/reviews strip → CTABanner), and retire
the obsolete `DienstenNav` scroll-spy (D-09). The hub is a clean router into the pillar
pages, not a content page — its rich per-service content now lives on the per-route pages
(seeded by 02-06).

## must_haves

- `app/diensten/page.tsx` renders a lean hub: hero + a 4-card grid from `pillars()` (each
  card links via `urlFor` to its pillar page) + a reviews/trust strip reusing the existing
  `ReviewCarousel` + `REVIEWS` (D-12) + `CTABanner`.
- The old anchored sections (`#wtw`, `#mechanisch`, `#airco`) and the `DienstenNav` import
  + usage are gone; no dead `#anchor` hrefs remain pointing at this page.
- `components/DienstenNav.tsx` is deleted (D-09) and nothing imports it.
- Basic `metadata` (title/description) retained for the hub (D-13).
- Design-system compliant (tonal layering, no `#000`, `Icon` wrapper, cheap motion only).
