# 02-05 — Taxonomy-derived navigation

## Objective

Make the Diensten navigation read the live taxonomy (IA-07, success criterion 5): the
desktop dropdown becomes a 4-pillar **mega-menu** (D-07) and the mobile drawer a 2-level
**accordion** (D-08), both sourced from `pillars()` + `childrenOf()`. Remove the hardcoded
`DIENSTEN_DROPDOWN`. Every service gets a sitewide internal link (a Phase-3 link-equity
asset). Depends on the routes (02-03) so every generated link resolves.

## must_haves

- Navbar Diensten panel renders 4 pillar columns from `pillars()`; each column header
  links to the pillar page (`urlFor`), each sub-item to its service page (`urlFor` over
  `childrenOf`). No `DIENSTEN_DROPDOWN` reference.
- MobileMenu Diensten entry expands to a 2-level accordion: pillar label links to the
  pillar page + a separate chevron toggles its sub-list (every sub reachable).
- `DIENSTEN_DROPDOWN` removed from `lib/constants.ts`; `NAV_LINKS`, `TARIEVEN_DROPDOWN`,
  `DropdownItem`, `SITE` unchanged.
- No hardcoded service nav entries remain anywhere (grep-clean).
